# Governance

Uni-CLI is an open-source project that welcomes contributions from everyone.

## Roles

### Maintainers

Maintainers have full commit access and are responsible for:

- Reviewing and merging pull requests
- Managing releases and versioning
- Setting project direction and priorities
- Enforcing the Code of Conduct

Current maintainers:

- **@ZenAlexa** — Project lead

### Committers

Committers have write access to the repository and can merge PRs in their area of expertise. They are nominated by maintainers based on sustained, high-quality contributions.

### Contributors

Anyone who submits a pull request, opens an issue, or improves documentation. All contributors are recognized in the CHANGELOG and release notes.

### Adapter Authors

Contributors who write and maintain adapters. Each adapter directory has an owner listed in `registry.json`. Adapter authors are the first point of contact for issues related to their adapters.

## Decision Making

- **Adapter additions**: Any well-tested adapter that follows the contribution guidelines can be merged.
- **Core changes**: Require review from at least one maintainer.
- **Breaking changes**: Require discussion in a GitHub issue before implementation.
- **Architecture decisions**: Documented in `docs/decisions/` as Architecture Decision Records (ADRs).

## Becoming a Committer

Sustained contributions over time (typically 5+ merged PRs) + nomination by a maintainer. There is no formal application process — just keep contributing quality work.
